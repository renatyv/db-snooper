Мне нужен скрипт который для MariaDB/MySQL/DuckDB/PostgreSQL генерирует .sql файлы с контекстом, которые потом можно пихать в контекст LLM для хорошей генерации запросов (text-to-sql)

Для каждой таблицы
1. Сгенерировать CREATE_TABLE со всеми индексами и констрейнтами
2. Статистика по каждой таблице:
* Если количество строк менее 50 - выбрать все
* Если более 20 строк
- по каждой колонке количество уникальных значений. Если менее 20, то выбрать все возможные
- минимальное, максимальное, медиана по каждой числовой колонке
- Топ 10 наиболее частых значений
3. Если непонятное название таблицы, спросить у пользователя интерактивно что это за таблица и зачем она

Если таблиц слишком много - разделить на несвязанные или слабо связанные домены. Так, чтобы не грузить все схемы в один контекст.
